# 凭证加密系统使用指南

## 系统概述

该系统实现了安全的凭证管理功能，支持：
- 📁 **默认占位图片** - 所有凭证默认显示统一的占位图
- 🔐 **服务端加密** - 使用不同密钥加密不同用户的凭证文件
- 🔑 **客户端解密** - 通过密码验证后自动解密并显示真实凭证
- 🔒 **权限控制** - 只有通过认证的用户才能查看对应的凭证

## 加密密钥

- **HJF凭证**: 使用 `ENCRYPT_HJF_KEY` 密钥加密
- **HJM凭证**: 使用 `ENCRYPT_HJM_KEY` 密钥加密

## 使用流程

### 1. 准备凭证图片

将原始凭证图片按照以下命名规则放入对应文件夹：

```
格式: voucher_YYYYMMDD_amount.扩展名
示例: voucher_20250115_1200.jpg
```

- **hjf** 的凭证放入 `vouchers/hjf/` 文件夹
- **hjm** 的凭证放入 `vouchers/hjm/` 文件夹

### 2. 运行加密脚本

```bash
node scripts/encrypt-vouchers.js
```

脚本会自动：
- 扫描原始图片文件
- 使用对应密钥加密
- 生成 `.enc` 加密文件
- 保存到 `vouchers/encrypted/` 文件夹

### 3. 更新数据配置

在 `assets/data/expenses-records.js` 中，将凭证字段设置为文件名：

```javascript
{
    id: 1,
    // ... 其他字段
    voucher: 'voucher_20250115_1200.jpg' // 只需文件名，不需要路径
}
```

### 4. 客户端使用

用户在浏览器中：

1. **查看列表** - 所有凭证显示为占位图片
2. **点击凭证** - 触发认证流程
3. **输入密码** - 验证用户身份
4. **自动解密** - 成功后显示真实凭证图片

## 认证机制

### 密码验证

系统复用现有的密码认证机制：
- **hjf** 和 **hjm** 使用各自的密码
- 密码验证成功后存储在客户端
- 支持交叉认证（根据数据量规则）

### 解密过程

1. **检查认证** - 验证用户是否已登录
2. **获取密码** - 从本地存储获取已验证的密码
3. **下载文件** - 获取加密的凭证文件
4. **解密显示** - 使用密码解密并显示图片

## 安全特性

### 服务端安全

- ✅ 原始图片不会部署到生产环境
- ✅ 只有加密后的 `.enc` 文件可访问
- ✅ 使用 AES-256-CBC 强加密算法
- ✅ 不同用户使用不同的加密密钥

### 客户端安全

- ✅ 密码验证后才能解密
- ✅ 解密过程在浏览器本地进行
- ✅ 支持缓存机制减少重复解密
- ✅ 页面卸载时自动清理敏感数据

### 权限控制

- ✅ 未认证用户只能看到占位图
- ✅ 认证用户只能查看自己的凭证
- ✅ 支持交叉认证（基于数据量规则）

## 开发指南

### 添加新凭证

1. **准备图片**：按命名规则保存到对应文件夹
2. **运行加密**：执行 `node scripts/encrypt-vouchers.js`
3. **更新数据**：在借款记录中添加凭证文件名
4. **测试验证**：确认加密解密流程正常

### 修改加密密钥

如需更改加密密钥：

1. **服务端**：修改 `scripts/encrypt-vouchers.js` 中的 `ENCRYPT_KEYS`
2. **客户端**：修改 `assets/js/voucher-decryptor.js` 中的 `ENCRYPT_KEYS`
3. **重新加密**：运行加密脚本处理所有文件

- GIF
- WebP
- BMP

## 故障排除

### 常见问题

**Q: 凭证显示"解密失败"**
A: 检查用户密码是否正确，文件是否存在且未损坏

**Q: 点击凭证没有反应**
A: 检查浏览器是否支持 WebCrypto API（需要 HTTPS 环境）

**Q: 加密脚本报错**
A: 确认 Node.js 版本支持，原始图片文件存在且格式正确

**Q: 占位图片不显示**
A: 检查 `vouchers/placeholder.svg` 文件是否存在

### 调试模式

开启浏览器控制台可查看详细的加密解密日志：

```javascript
// 查看认证状态
console.log(window.AuthHelper.isAuthenticated('hjf'));

// 查看解密器状态
console.log(window.VoucherDecryptor.isWebCryptoSupported());

// 清理图片缓存
window.VoucherDecryptor.clearImageCache();
```

## 部署注意事项

### 生产环境

1. **文件排除**：确保 `vouchers/hjf/` 和 `vouchers/hjm/` 不被部署
2. **HTTPS必需**：WebCrypto API 要求 HTTPS 环境
3. **密钥保护**：加密密钥应妥善保管，不要泄露
4. **备份策略**：定期备份原始图片和加密文件

### 性能优化

- 图片缓存机制减少重复解密
- 占位图片使用轻量级 SVG 格式
- 按需解密，不预加载所有图片
